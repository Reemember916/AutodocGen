# PROJECT RS422 Maintenance Host

Python 图形化上位机，用于调试本项目维护 RS422 通信。

## 功能

- 串口打开/关闭与端口刷新。
- 默认串口参数：`115200 8O1`。
- 发送 16 字节维护帧，自动计算校验。
- 支持三类已知指令：
  - 通信状态信息：`0x11`
  - 地面维护功能：`0x99`
  - 地面控制：`0x66`
  - RIU429模拟注入：`0x67`
  - 状态机条件注入：`0x68`
- 支持手动 16 字节帧编辑，自动重写 `EB 90` 帧头和 checksum。
- 接收并解析 DSP 回传 `55 AA` 帧。
- 支持 ID0 基础状态帧、ID1 主备轮值诊断帧、ID2 输出授权诊断帧、ID3 BIT/故障摘要帧、ID4 通信来源/余度来源帧解析。
- 右侧实时状态页签显示控制总览、输出授权、BIT故障、余度来源、RIU模拟状态和状态条件注入状态。
- 支持模拟接收帧，无硬件时也能验证解析显示。
- 支持保存日志。

## 安装依赖

Tkinter 通常随 Python 自带。真实串口通信需要安装 `pyserial`：

```bash
python3 -m pip install pyserial
```

未安装 `pyserial` 时，GUI 仍可启动，但只能使用模拟接收功能。

## 运行

在项目根目录执行：

```bash
python3 tools/rs422_host/rs422_host_gui.py
```

## 协议摘要

上位机发送到 DSP：

```text
EB 90 [frame_count] [cmd] ... [maint_state] [reserved] [checksum]
```

DSP 回传到上位机：

```text
55 AA [frame_id] ...
```

固定帧长：16 字节。

校验：

```text
checksum = (~sum(bytes[0:15]) + 1) & 0xFF
```

## 常用指令

通信状态信息 `0x11`：

- Byte 4 低 4 bit：429 通信信息 ID。
- Byte 4 高 4 bit：流量测量盒 422 信息 ID。
- Byte 5：429 接收 Label。
- Byte 6-8：读取地址，低字节在前。
- Byte 9：余度信息索引。
- Byte 10：软件版本信息索引。
- Byte 11：模拟量信息索引。
- Byte 12：泵控制器信息索引。
- Byte 13：维护状态码，默认 `0x22`。

地面维护功能 `0x99`：

- Byte 4：功能码。
- 功能码 `0x02` 数据下载时：
  - Byte 5-8：下载起始地址，低字节在前。
  - Byte 9-12：下载长度，低字节在前。

地面控制 `0x66`：

- Byte 4：更新标志，默认 `0x34`。
- Byte 5：维护指令 ID。
- Byte 6-7：控制数据，低字节在前。

RIU429模拟注入 `0x67`：

- Byte 4：子命令。
- Byte 5-12：子命令载荷。
- Byte 13：维护状态码，固定 `0x22`。
- 子命令：
  - `0x00`：关闭模拟。
  - `0x01`：开启模拟，Byte5为超时秒，Byte6为RIU通道。
  - `0x10`：加受油指令，包含对象、模式、轮载、维护命令、控制命令。
  - `0x11`：预设油量和全机总油量，单位0.1L。
  - `0x12`：0-3号油箱油量，单位0.1L。
  - `0x13`：4号油箱、燃油密度、指示空速。
  - `0x14`：阀、泵、高低液位和故障字。

状态机条件注入 `0x68`：

- Byte 4：子命令。
- Byte 5-12：子命令载荷。
- Byte 13：维护状态码，固定 `0x22`。
- 子命令：
  - `0x00`：关闭全部条件注入。
  - `0x01`：开启条件注入，Byte5为超时秒。
  - `0x10`：IO条件覆盖，Byte5为覆盖位图，Byte6为维护IO，Byte7为28V电源IO。
  - `0x20`：BIT条件覆盖，Byte5为覆盖位图，Byte6为PuBIT关键故障，Byte7为IFBIT等级，Byte8为MBIT等级。

## 回传帧

ID0 基础状态：

- 系统时间。
- 系统状态 / 上一拍系统状态。
- 工作模式 / 上一拍工作模式。
- 运行期控制权 / 静态主备身份。
- 控制输出有效、CHV 资格、通道号、控制功能、CHV 输入快照。

ID1 主备轮值诊断：

- 本地期望主通道。
- 对端期望主通道。
- 仲裁主通道。
- 通道类型判别结果码。

ID2 输出授权诊断：

- 运行期控制权、静态主备身份、本通道 ID。
- 本端 CHV 资格、控制输出状态。
- CHV 输入原始字及 myCHV / otherCHV / WDV / CPUV / LATCH_EN。
- 对端在线、是否看到对端控制权。
- 输出条件位图：本端主控、CHV资格、CHV回绕、输出有效、对端在线、对端持权。

ID3 BIT/故障摘要：

- PuBIT 关键故障标志：`0` 无关键故障，`1` 存在关键故障。
- IFBIT 故障等级、MBIT 故障等级。
- 控制故障位图：bit0 通信故障、bit1 测量故障、bit2 不平衡故障、bit3 综合故障。
- 控制故障原因码。
- IFBIT 低 32 位故障签名。
- MBIT 低 16 位故障签名。

ID4 通信来源/余度来源：

- 通信来源原始字：低 2 bit 为 RIU，bit2-3 为 CCDL，bit4-5 为 KZZZ。
- RIU、左 KZZZ、右 KZZZ、CCDL 的余度状态。
- RIU 心跳、左右 KZZZ 快照、对端系统状态、对端通道类型和对端期望主通道。
- 来源有效位图：bit0 RIU有效、bit1 左KZZZ有效、bit2 右KZZZ有效、bit3 CCDL有效、bit4 RIU选源无效、bit5 CCDL选源无效、bit6 KZZZ选源无效。

ID5 RIU模拟状态：

- RIU模拟开启状态、目标RIU通道、最近子命令。
- 剩余超时时间，单位100ms。
- 模拟对象、模式、轮载、总油量和故障字。
- 状态位图：bit0模拟开启、bit1超时有效、bit2目标通道有效。

ID6 状态机条件注入：

- 条件注入开启状态、最近子命令、剩余超时时间。
- 维护IO、28V电源IO、PuBIT关键故障、IFBIT等级、MBIT等级覆盖状态。
- 状态位图：bit0注入开启、bit1超时有效、bit2 IO覆盖、bit3 BIT覆盖。
