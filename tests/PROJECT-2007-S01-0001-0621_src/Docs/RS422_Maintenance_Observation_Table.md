# RS422维护通信观测表

本文档按当前代码整理，便于单板调试时直接对照上位机或串口助手数据。

代码来源：

- `Src/Application/Communication/MaintDataSendPack.c`
- `Include/Application/Communication/Comm422.h`
- `Src/Application/Communication/Comm422.c`

## 发送帧通用格式

维护发送帧固定为 16 字节。

| 字节 | 含义 | 说明 |
| --- | --- | --- |
| Byte0 | 帧头1 | 固定 `0x55` |
| Byte1 | 帧头2 | 固定 `0xAA` |
| Byte2 | 包号 | 当前为 `0x00` 到 `0x07` |
| Byte3 | 帧计数 | 每个包号各自递增，低8位循环 |
| Byte4~Byte14 | 数据区 | 各包号含义不同 |
| Byte15 | 校验 | Byte0~Byte14 累加后取反加1，保留低8位 |

当前一次打包 ID0~ID6 共 7 包，发送函数每次发 1 包，按 ID0、ID1、...、ID6 轮发。

## ID0 基础状态

| 字节 | 含义 | 是否自动变化 |
| --- | --- | --- |
| Byte4~Byte7 | 系统时间 `sysTime()`，低字节在前 | 是 |
| Byte8 | 低4位当前系统状态，高4位上一拍系统状态 | 状态变化时变化 |
| Byte9 | 低4位当前工作模式，高4位上一拍工作模式 | 模式变化时变化 |
| Byte10 | 低4位运行期角色，高4位静态主备身份 | 角色/判型变化时变化 |
| Byte11 | bit0 输出有效，bit1 本端CHV资格，bit3 本端通道号；高4位 `OilMode` | 条件变化时变化 |
| Byte12 | 低4位当前控制功能，高4位上一拍控制功能 | 状态机跳转时变化 |
| Byte13 | CHV输入快照低8位 | CHV输入变化时变化 |
| Byte14 | 空中加油结束状态 | 流程结束状态变化时变化 |

单板无 RIU 时，最容易观察的自动变化字段是 Byte3 和 Byte4~Byte7。

## ID1 主备轮值诊断

| 字节 | 含义 | 是否自动变化 |
| --- | --- | --- |
| Byte4 | 本端期望主通道 `localPreferredMasterChId` | 判型/轮值变化时变化 |
| Byte5 | 对端期望主通道 `peerPreferredMasterChId` | 对端数据变化时变化 |
| Byte6 | 仲裁主通道 `arbMasterChId` | 仲裁变化时变化 |
| Byte7 | 启动判型结果码 `ChTypeCode` | 判型变化时变化 |
| Byte8~Byte14 | 保留 | 当前为0 |

用于定位 `runtimeRole` 卡在备份的问题，建议和 ID0 Byte10、ID2 Byte4/Byte5 一起看。

## ID2 输出授权诊断

| 字节 | 含义 | 是否自动变化 |
| --- | --- | --- |
| Byte4 | 低4位运行期角色，高4位静态主备身份 | 角色变化时变化 |
| Byte5 | 低4位本通道ID，高4位启动判型结果码 | 判型变化时变化 |
| Byte6 | 本端CHV资格 `localChvPermit` | 条件变化时变化 |
| Byte7 | 控制输出状态 `conOutState` | 输出授权变化时变化 |
| Byte8~Byte9 | CHV输入快照，低字节在前 | CHV输入变化时变化 |
| Byte10 | 对端存活 `peerAlive` | 对端链路变化时变化 |
| Byte11 | 对端控制数据已见 `peerCtrlSeen` | 对端控制帧变化时变化 |
| Byte12 | 低4位系统状态，高4位工作模式 | 状态变化时变化 |
| Byte13 | 低4位控制功能，高4位维护功能 | 功能切换时变化 |
| Byte14 | 输出授权条件位图 | 条件变化时变化 |

Byte14 位定义：

| 位 | 含义 |
| --- | --- |
| bit0 | 本端为运行期主控 |
| bit1 | 本端CHV资格有效 |
| bit2 | 本端CHV输入有效 |
| bit3 | 控制输出状态有效 |
| bit4 | 对端存活 |
| bit5 | 已见对端控制数据 |

## ID3 BIT与控制故障摘要

| 字节 | 含义 | 是否自动变化 |
| --- | --- | --- |
| Byte4 | PuBIT关键故障标志，0无关键故障，1存在关键故障 | PuBIT变化时变化 |
| Byte5 | IFBIT故障等级 | 周期BIT变化时变化 |
| Byte6 | MBIT故障等级 | 维护BIT变化时变化 |
| Byte7 | 控制故障位图 | 故障判定变化时变化 |
| Byte8 | 控制故障原因码 `reason` | 故障原因变化时变化 |
| Byte9~Byte12 | IFBIT低32位，低字节在前 | 周期BIT变化时变化 |
| Byte13~Byte14 | MBIT低16位，低字节在前 | 维护BIT变化时变化 |

Byte7 位定义：

| 位 | 含义 |
| --- | --- |
| bit0 | 通信故障 |
| bit1 | 测量故障 |
| bit2 | 油量不平衡故障 |
| bit3 | 存在任一控制故障 |

| Byte6 | 左KZZZ当前时间请求状态 | KZZZ链路变化时变化 |
| Byte7 | 右KZZZ当前时间请求状态 | KZZZ链路变化时变化 |
| Byte8 | CCDL系统状态数据状态 | 板间链路变化时变化 |
| Byte9 | RIU心跳数据低8位 | RIU数据变化时变化 |
| Byte10 | 左KZZZ当前时间请求低8位 | KZZZ数据变化时变化 |
| Byte11 | 右KZZZ当前时间请求低8位 | KZZZ数据变化时变化 |
| Byte12 | CCDL系统状态低8位 | 板间数据变化时变化 |
| Byte13 | 低4位CCDL静态主备身份，高4位CCDL期望主通道 | 板间数据变化时变化 |
| Byte14 | 关键来源有效位图 | 来源状态变化时变化 |

Byte14 位定义：

| 位 | 含义 |
| --- | --- |
| bit0 | RIU心跳数据非错误 |
| bit1 | 左KZZZ数据非错误 |

| bit6 | KZZZ来源无效 |

## ID5 版本、CRC与维护执行结果

| 字节 | 含义 | 是否自动变化 |
| --- | --- | --- |
| Byte4~Byte5 | DSP软件版本，低字节在前 | 固定或版本变化 |
| Byte6~Byte7 | 硬件版本，低字节在前 | 硬件版本调整后变化 |
| Byte8~Byte9 | 应用软件CRC，低字节在前 | 执行软件CRC命令后变化 |
| Byte10~Byte11 | 加载/更新软件CRC，低字节在前 | 当前代码预留上报 |
| Byte12 | 最近一次维护功能码 | 执行维护命令后变化 |
| Byte13 | 最近一次维护功能执行结果 | 执行维护命令后变化 |
| Byte14 | 状态位图 | 条件变化时变化 |

Byte13 执行结果：

| 值 | 含义 |
| --- | --- |
| `0x00` | 未执行 |
| `0x01` | 执行成功 |
| `0x02` | 暂不支持 |
| `0x03` | 参数无效 |
| `0x04` | 执行失败 |

Byte14 位定义：

| 位 | 含义 |
| --- | --- |
| bit0 | 硬件版本读取有效 |
| bit1 | 应用软件CRC非0 |
| bit2 | 更新软件CRC非0 |
| bit3 | 最近维护命令执行成功 |
| bit4 | 特定数据区存在待写状态 |

这是验证 422 维护功能最关键的一包。软件CRC、硬件版本调整、BIT清除、记录擦除后，都先看 Byte12/Byte13。

## ID6 完整BIT位图

| 字节 | 含义 | 是否自动变化 |
| --- | --- | --- |
| Byte4~Byte5 | PuBIT结果，低字节在前 | 上电BIT变化时变化 |
| Byte6~Byte9 | MBIT低32位，低字节在前 | 维护BIT变化时变化 |
| Byte10~Byte13 | IFBIT低32位，低字节在前 | 周期BIT变化时变化 |
| Byte14 | 低4位IFBIT等级，高4位MBIT等级 | BIT等级变化时变化 |

ID6 用于补足 ID3 的摘要信息。若想确认 BIT 清除是否生效，先看 ID6 的 MBIT/IFBIT 位图是否清零，然后再看故障是否按检测周期重新置位。

## 维护命令功能码

维护命令进入后，执行结果通过 ID5 Byte12/Byte13 回传。

| 功能码 | 含义 | 观测点 |
| --- | --- | --- |
| `0x01` | 软件CRC计算 | ID5 Byte8~Byte9、Byte12、Byte13 |
| `0x02` | 数据下载 | 维护口发送下载数据流；ID5 Byte12、Byte13 |
| `0x03` | 信息擦除 | 擦除周期记录区；ID5 Byte12、Byte13 |
| `0x04` | 硬件版本调整 | ID5 Byte6~Byte7、Byte12、Byte13 |
| `0x05` | BIT信息清除 | ID6 BIT位图、ID5 Byte12、Byte13 |

硬件版本调整参数按当前代码解释：

| 字节 | 含义 |
| --- | --- |
| Byte4 | 功能码 `0x04` |
| Byte5 | 硬件版本低字节 |
| Byte6 | 硬件版本高字节 |
| Byte7 | 校验值 |

校验值计算口径：`~((低字节 + 高字节) & 0xFF) & 0xFF`。

## 单板调试建议

不接 RIU 时也能稳定观察这些自动变化字段：

| 观测字段 | 预期现象 |
| --- | --- |
| 任意ID Byte3 | 对应包号每次重新打包后递增 |
| ID0 Byte4~Byte7 | 系统时间持续递增 |
| ID0 Byte8~Byte12 | 状态机、模式、角色、控制功能变化时跳变 |
| ID3 Byte5~Byte8 | BIT等级和控制故障变化时跳变 |
| ID5 Byte12~Byte13 | 维护命令执行后显示最近功能码和执行结果 |
| ID6 Byte6~Byte13 | BIT故障出现、恢复或清除后变化 |

若只想确认“422有没有持续刷新”，优先看任意包 Byte3 和 ID0 Byte4~Byte7。
