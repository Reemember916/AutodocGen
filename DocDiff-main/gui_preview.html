建议怎么调布局

  • 按任务分组：对比对象 / 问题单 / 文档选项，减少「一长串勾选」的压迫感。
  • 主路径三行固定：旧、新、输出永远在最上方，不随模式乱跳位置。
  • 问题单独立成块：台账、前缀、匹配策略、自动匹配放一起；LLM 设置进二级对话框，避免主界面堆 Key。
  • 模式切换只显隐整块：docx 显示 Fuzzy+元数据+表格；code 显示 C 省略行，不要逐个控件 disable 散落。
  • 日志占满剩余高度：上面表单可滚动或折叠，下面日志 expand=True 一直吃空间。
  • 可选后续：生成后弹「匹配结果」小表（序号/问题/单号/挂到章节/分数），确认后再写 Word——准确率与可归档性会好一截。

实施优先级（若改 GUI）

  1. P0：分组 LabelFrame + 窗口可拉高日志区（不动业务逻辑）
  2. P1：匹配策略下拉（rules/hybrid/llm）+ LLM 设置对话框
  3. P2：匹配结果预览表 / 打开 dump JSON
