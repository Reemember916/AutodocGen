CSCI 详细设计生产工作台 V2.0

本工具从 C/C++ 源码自动生成 CSCI 详细设计 Word 文档，支持新建与维护两种工作模式。

功能概览

  • 从 C/C++ 源码生成 CSCI 详细设计 Word 文档（.docx）。
  • 支持单文件与工程模式，工程模式可按层级（应用层 / 中间层 / 驱动层）与自定义顺序生成。
  • 支持维护已交付文档：CSU 重编与代码差异更新。
  • 术语一致性检查：检测跨工程中文命名不一致与符号字典冲突。
  • 支持 AI 辅助（可选），AI 只做受控表达增强，源码事实由规则、AST 与 LSP 约束。
  • 提供日志、步骤进度与可续跑能力。

新建设计文档流程

  1. 进入 新建设计文档 页面，选择 单 C 文件 或 工程目录。
  2. 工程模式：点击 加载，展开应用层 / 中间层 / 驱动层文件树，拖拽排序。
  3. 填写 输出 Word 路径（.docx），可选择模板与已批准的审查决策文件。
  4. 展开 高级生成选项 可设置需求 ID 前缀、是否启用 AI 等。
  5. 点击底部固定主按钮 生成 Word 详细设计，预检摘要会显示输入、输出、预计函数数与风险。
  6. 运行时切换为步骤面板、实时日志、当前函数、进度与预计剩余时间。
  7. 完成后在结果区展示 Word、审查包、调用图、失败清单等产物，可 打开、打开所在目录 或 查看结果。

在 1366×768 分辨率下，首个主操作按钮无需滚动即可点击。

维护已有文档流程

进入 维护已有文档 页面，选择维护方式：

CSU 重编

  • 选择源 Word 文档、源码文件、目标 CSU 标识。
  • 支持单项和批量重编（批量时在文本区逐行填写函数名与 CSU）。
  • 生成前显示将替换的文档范围，覆盖已有输出副本时会二次确认。
  • 默认不覆盖原始 Word，输出为副本。

代码差异更新

依序填写旧代码、新代码、旧 Word、输出 Word 副本。三种更新模式：

模式说明是否修改输出文档
仅生成更新计划分析代码差异，输出更新计划 JSON 与审查 HTML，不修改 Word 文档否
自动应用低风险更新自动应用低风险差异到 Word 副本，高风险项保留待人工决定是（低风险部分）
按审查决定更新需导入审查决定文件，按决定逐项应用更新是（按决定）

plan-only 模式结束后，主结果是更新计划和审查页，而非"完成"的 Word 文档。请在结果区打开对应产物。

术语与一致性

  • 选择工程后执行一致性检查，面板展示评分、总符号数、一致数、不一致数与字典冲突数。
  • 不一致明细列表显示符号、变体、严重程度与建议术语。
  • 点击 导出报告 保存 JSON 格式的完整一致性报告。
  • 点击 打开术语表处理 引导你在术语表或符号字典中手动修正对应条目（自动修复尚未实现）。

设置说明

基本规则

选项说明
需求 ID 前缀生成的需求编号前缀
完成后自动打开生成后自动打开输出文档
排除目录工程扫描时跳过的目录名称（每行一个）
分层规则中间层 / 驱动层识别关键词（逗号分隔）
术语表覆盖术语映射，用于命名优化
符号字典覆盖精确符号映射，用于局部变量、全局变量、结构体成员、函数名、宏名的一致化

AI 连接

选项说明
AI 模式关闭 / 开启。开启后 AI 只作为受控表达增强，源码事实仍由规则、LSP 与 AST 约束
调用方式本地、DeepSeek 或其他 OpenAI 兼容远程模型接口
模型切换 AI 提供方后点击 刷新模型列表，后台异步加载，界面不会冻结
Context(num_ctx)上下文长度（依模型而定）
AI 并发数开启 AI 后用于函数级并发生成，默认 1，建议从 1 开始调优
Proxy可选代理地址

断网或切换 AI 提供方时主窗口保持可操作。模型列表加载为后台异步任务。

性能

  • 工程扫描预筛选：跳过无注释 / 无函数文件，提高扫描速度。
  • 并行预处理线程：0 为自动。
  • 日志采样间隔：减少日志刷屏。

高级参数

  • max_dist / min_ratio：AI JSON key 容错参数。
  • ai_retry_times：AI 失败重试次数。
  • ai_fail_policy：失败策略（fallback / skip_function / circuit_fallback）。
  • ai_regression_rounds：首轮效果不佳时的自动回归补跑轮数，默认 1。

实验室

实验室页面隔离尚未验证或缺少后端能力支撑的实验性功能。当前包含：

  • MD → C（从 Markdown 逆向生成 C 代码）：标记为"实验性"，在缺少后端实现时禁用并说明原因。

实验室中的功能不保证可用，不放入主流程导航。

日志与诊断

帮助与诊断页面包含两个标签：

  • 使用帮助：即本手册。
  • 运行诊断：显示环境信息、版本号、Python 路径等；可导出日志、清空日志、查看诊断日志。

日志默认最多保留 5000 行，超出后自动丢弃旧日志。失败函数记录可导出为错误报告。

常见问题

提示"工程目录不存在 / 未找到 SRC"

请确认工程目录正确，且包含 SRC（不区分大小写）。

提示"输出文件必须是 .docx"

请在输出路径中选择 .docx 扩展名。

AI 调用失败或很慢

检查模型列表是否已刷新、提供方地址是否可访问。可先将 AI 模式切换为关闭，使用纯规则生成。

维护模式下覆盖已有输出时提示确认

维护操作默认不覆盖原始 Word 文档，输出为副本。覆盖已有输出副本时会弹出二次确认对话框。

术语一致性面板显示不一致但没有"自动修复"按钮

自动修复功能因无法解释修改范围而未启用。请根据不一致明细在术语表或符号字典中手动修正，或导出报告后参照处理。

plan-only 模式结束后看不到"完成"的 Word

这是设计行为。plan-only 模式只生成更新计划和审查页，不修改 Word 文档。请在结果区打开更新计划 JSON 和审查 HTML。
